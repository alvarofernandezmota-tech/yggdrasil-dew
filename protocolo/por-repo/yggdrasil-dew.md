# Protocolo yggdrasil-dew

## Contexto rápido

- Repo de trabajo activo del ecosistema
- **Toda la gestión de trabajo va aquí** (issues, protocolos, estado)
- Es el único repo donde se crean issues de trabajo

## Convención de issues

| Prefijo | Tipo | Prioridad |
|---|---|---|
| `HAL-XXX` | Bug bloqueante / incidente | P0 |
| `AUDIT-XXX` | Auditoría / revisión | P1 |
| `FEAT-XXX` | Nueva funcionalidad | P2 |
| `DOCS-XXX` | Documentación | P3 |

## Antes de crear un issue

1. Verificar que no existe ya uno similar abierto
2. Usar el prefijo correcto
3. Asignar prioridad en el título
4. Linkar repos e islas afectadas

## Antes de cerrar un issue

1. Comentar qué se hizo exactamente
2. Linkar commits o PRs relacionados
3. Indicar si quedan sub-tareas abiertas (crear issues hijos)

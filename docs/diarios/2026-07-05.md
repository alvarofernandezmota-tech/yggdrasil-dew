### 🗺️ Arquitectura de retroalimentación GitHub ↔ Obsidian

El ecosistema tiene que funcionar así:

```
Obsidian (vault local)
  ├── Lee WIKI---PERSONAL vía git pull
  ├── Lee yggdrasil-dew/docs/ vía git pull
  └── Escribe notas → git push → GitHub

GitHub (repos)
  ├── yggdrasil-dew = cerebro técnico (issues, roadmap, diarios)
  ├── WIKI---PERSONAL = mapa conceptual personal (islas, conocimiento)
  └── Cada repo opera = fuente de verdad de su dominio

Issues = tareas accionables
  ├── Cada issue tiene repo correcto (no flotar en WIKI)
  ├── WIKI no gestiona tareas — gestiona conocimiento
  └── dew gestiona tareas técnicas y arquitectura
```

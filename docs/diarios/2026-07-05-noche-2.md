## 💡 Decisiones de arquitectura tomadas esta sesión

### La regla definitiva Dew vs WIKI

```
¿Hace que el sistema funcione mejor?        → Dew / repo técnico
¿Me ayuda a pensar/navegar mi conocimiento? → WIKI / isla
```

### Tipos de isla (canon)
| Tipo | Descripción |
|---|---|
| `meta` | Gobierna o da contexto a todo el ecosistema |
| `técnica` | Área con repos, código y operaciones propias |
| `personal` | Contexto humano: vida, conocimiento, identidad |

### Niveles de madurez de isla (canon)
| Nivel | Descripción |
|---|---|
| `estable` | Documentada, activa, sin deuda estructural |
| `activo-con-deuda` | Funciona pero tiene deuda técnica conocida |
| `borrador` | Esqueleto listo, contenido pendiente |
| `legacy` | Archivado, ya no se desarrolla activamente |

### Lo que va en WIKI vs Dew (tabla definitiva)
| Dew | WIKI |
|---|---|
| Arquitectura técnica del ecosistema | Islas conceptuales (mapas mentales) |
| Roadmaps, backlog, deuda técnica | Relaciones entre islas |
| Agentes, workflows, scripts, MCP | Contexto personal y hardware como mapa |
| Convenciones del ecosistema | Vida personal (VIDAPERSONAL) |
| Diarios de sesión técnica | — |
| Issues técnicos | Issues solo de estructura WIKI |

### Si una isla tiene procedimientos técnicos
> Los procedimientos se van a Dew. La isla se queda como mapa conceptual que enlaza.

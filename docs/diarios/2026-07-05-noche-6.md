## 🔴 Diagnóstico: tailscale_monitor crashloop

### Log real
```
[*] Tailscale Monitor starting...
[*] Tailscale Monitor starting...
[*] Tailscale Monitor starting...
... (>100 líneas idénticas en segundos)
```

### Diagnóstico

El contenedor arranca, ejecuta la línea de inicio del script de Python/bash, y **muere inmediatamente antes de hacer nada**. Docker lo reinicia. El ciclo se repite cada pocos segundos.

Causas más probables (por orden de probabilidad):
1. **Variable de entorno faltante** — el script espera `TAILSCALE_API_KEY` u otra var que no existe en el entorno Docker
2. **Dependencia de red no disponible** — el script intenta conectar a Tailscale API y falla si no hay red/token
3. **Import de Python fallando** — algún módulo no instalado en la imagen (falla silenciosa antes del try/except)
4. **Archivo de configuración no montado** — volumen roto o ruta incorrecta en docker-compose

### Comandos para diagnosticar en profundidad

```bash
# Ver el error REAL (el crashloop oculta el traceback)
docker run --rm --entrypoint sh $(docker inspect tailscale_monitor --format '{{.Config.Image}}') -c 'python monitor.py 2>&1' || true

# Ver variables de entorno del contenedor
docker inspect tailscale_monitor | python3 -c "import sys,json; d=json.load(sys.stdin)[0]; [print(e) for e in d['Config']['Env']]"

# Ver el docker-compose del servicio
cd ~/yggdrasil-secops && cat docker-compose.yml | grep -A 20 tailscale_monitor

# Ver el script que ejecuta
docker cp tailscale_monitor:/app/monitor.py /tmp/monitor.py && head -30 /tmp/monitor.py

# Parar el crashloop mientras investigas
docker stop tailscale_monitor
```

### Fix más probable (si es variable de entorno)

```bash
# Verificar qué variables necesita
cat ~/yggdrasil-secops/.env | grep -i tailscale
# o
cat ~/yggdrasil-secops/docker-compose.yml | grep -A5 environment
```

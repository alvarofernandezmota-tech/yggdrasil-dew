## ✅ HAL-004 RESUELTO — tailscale_monitor

### Causa raíz real

**`pgrep` no estaba instalado en la imagen `python:3.11-slim`.**

El healthcheck del `docker-compose.maestro.yml` ejecuta:
```
pgrep -f tailscale_monitor || exit 1
```

Pero `python:3.11-slim` no incluye `procps` (el paquete que provee `pgrep`).
Resultado: el healthcheck devolvía siempre:
```
/bin/sh: 1: pgrep: not found  → ExitCode: 1
```

El script `tailscale_monitor.py` **funcionaba perfectamente**. Los 4 nodos estaban configurados y siendo monitorizados. El problema era únicamente el healthcheck.

### Evidencia

```
Last log: {'ExitCode': 1, 'Output': '/bin/sh: 1: pgrep: not found\n'}
OCI runtime exec failed: exec failed: unable to start container process:
  exec: "pgrep": executable file not found in $PATH
```

### Fix aplicado

Commit [e4447b1](https://github.com/alvarofernandezmota-tech/yggdrasil-secops/commit/e4447b1dcee1965d1d9d55aebe0446c57e1c75c0) en `yggdrasil-secops`:

```dockerfile
# Antes
RUN apt-get update && apt-get install -y --no-install-recommends \
    iputils-ping \
    && rm -rf /var/lib/apt/lists/*

# Después
RUN apt-get update && apt-get install -y --no-install-recommends \
    iputils-ping \
    procps \
    && rm -rf /var/lib/apt/lists/*
```

### Rebuild en Madre

```
git pull origin main
docker compose -f docker-compose.maestro.yml build tailscale_monitor  → 295s
docker compose -f docker-compose.maestro.yml up -d tailscale_monitor  → Started
```

**Estado esperado tras 120s:** `(healthy)`

### Aprendizaje canon

> Cuando se usa `python:X-slim` como imagen base, los healthchecks que usen
> `pgrep`, `ps`, `top`, `kill` requieren instalar `procps` explícitamente.
> Alternativa más robusta: usar `pidof` (incluído en `sysvinit-utils`) o
> healthcheck HTTP si el servicio expone un endpoint.

### Problema conceptual: tailscale_monitor

El bot hace `ping ICMP` a las IPs de Tailscale desde Docker con `network_mode: host`.
Esto es **fragil** porque:
1. El daemon `tailscaled` vive fuera del contenedor
2. Un ping a `100.x.x.x` puede fallar aunque Tailscale esté OK
3. Lo correcto sería consultar `tailscale status --json` directamente

**Acción futura:** reescribir el monitor para usar `tailscale status --json`
instead of ICMP pings. Crear issue en `yggdrasil-secops`.

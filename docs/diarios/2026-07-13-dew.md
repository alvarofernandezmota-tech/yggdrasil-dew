## Investigación Gemini — 2026-07-13

> Auditoría externa recibida de Gemini sobre el estado de producción del homelab.
> Estado: **backlog — ningún punto confirmado como ejecutado todavía.**
> Contraste con el Plan Maestro: las propuestas son válidas y complementarias, no contradicen la arquitectura.

### 1. SPOF identificado: `.env` en Madre

- El `.env` es un Punto Único de Fallo. Si se corrompe, n8n → LiteLLM → toda la cadena muere.
- **Propuesta:** tratar los secretos como contratos de estado, no como configuración estática.
  Herramientas candidatas: `sops` + age key, o Vault (más complejo).
- **Alineado con:** issue #44 ya abierto. Acción adicional: crear `.env.template` con variables obligatorias.

### 2. Gaps de documentación detectados

- Falta `docs/secrets.md` — ciclo de vida de rotación de tokens sin tocar historial Git.
- Falta `docs/runbooks/` — procedimientos de emergencia. **→ Creado hoy: `docs/runbooks/DRP.md`** (esqueleto, requiere completar con datos reales de infra).

### 3. Semántica Conocimiento / Formación / Vida

- Gemini propone separación clara ya alineada con las islas existentes:
  - **Conocimiento** = mapa de verdad estático (Wiki)
  - **Formación** = bitácora de procesos y experimentos (sesiones diarias)
  - **Vida** = diario personal, salud, mantenimiento físico (VIDAPERSONAL)
- **Recomendación pendiente:** mover `GRAFO-ECOSISTEMA.md` a la raíz de la Wiki como `index.md`.

### 4. Risk Assessment adicional

- HDD 28k horas (#31): Gemini añade → implementar `btrfs scrub` semanal por cron, no solo monitorizar.
- Docker-compose drift (#43): "versión 16" indica cambios manuales sin PR. Esto rompe GitOps.
  **Norma a establecer:** ningún cambio en producción sin pasar por PR + GitHub Actions.

### 5. Operabilidad — lo que faltaba

- **Healthchecks en docker-compose.yml:** el sistema no sabe si un contenedor está vivo o solo corriendo.
  Ejemplo para `log_guardian`: `healthcheck: test: curl -f http://localhost:8080/health`
- **DRP documentado:** ¿cuánto tiempo para recuperar el RAG/VectorDB si Madre muere? → ver `docs/runbooks/DRP.md`
- **Agregación de logs:** múltiples servicios, logs dispersos. Propuesta: Promtail + Loki como capa de agregación.

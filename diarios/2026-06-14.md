### ~21:28–23:35 CEST — Fix Groq 401 + upgrade modelo

**Síntoma**: NLP completamente bloqueado. Todos los mensajes de texto libre fallaban:
```
ERROR — Error clasificando intent: Error code: 401 - {'error': {'message': 'Invalid API Key'}}
```

**Diagnóstico**:
1. `docker compose exec bot env | grep GROQ` → variable existía en contenedor
2. Verificación desde Google Colab con `requests.get` a `https://api.groq.com/openai/v1/models`
3. Key antigua → `401`. Key nueva → `200` con lista de modelos ✅
4. Modelo `llama3-8b-8192` — **deprecado** en junio 2026, no aparece en catálogo

**Causa raíz**: key expuesta/revocada + modelo deprecado.

**Fixes aplicados**:
- Key rotada en [console.groq.com](https://console.groq.com), nueva en `.env`
- `.env` verificado en `.gitignore` — nunca llega a GitHub ✅
- Modelo actualizado: `llama3-8b-8192` → `llama-3.3-70b-versatile` (128k contexto)
- Recarga forzada: `docker compose down && docker compose up -d`
  > `docker restart` NO recarga `.env`. Solo `down + up` funciona.

## Aprendizajes del día

| Aprendizaje | Detalle |
|---|---|
| `docker restart` vs `down/up` | `restart` NO recarga `.env`. Siempre `down && up` al cambiar variables |
| Verificar API keys aislada | Google Colab + `requests` para aislar si el problema es la key o Docker |
| Modelos Groq deprecados | `llama3-8b-8192` eliminado. Usar `llama-3.1-8b-instant` o `llama-3.3-70b-versatile` |
| `git pull --rebase` | Evita merge commits cuando el remote tiene commits nuevos |
| lan-mouse requiere Wayland | No funciona via SSH sin display activo |
| AP Isolation | Bloquea UDP entre dispositivos de la misma red WiFi — problema común en routers domésticos |

## 💡 Decisión clave — Por qué workflows resuelve el problema de fondo

El problema real: el repo siempre está desactualizado porque actualizar a mano tiene fricción.

La solución tiene **dos capas**:
1. **Obsidian** (HOY) → edición con cero fricción del vault
2. **GitHub Actions / Workflows** (próxima semana) → automatización: commits nocturnos, diario automático

Combinados = el repo se mantiene vivo sin depender de una IA.

```
Workflow orden de aprendizaje:
1. Obsidian + plugin Git  → HOY (instalar en varopc)
2. Scripts bash           → YA LISTOS (audit-repo.sh, daily-sync.sh)
3. GitHub Actions diario  → próxima semana
4. n8n                    → este mes
5. thdora /diario         → este mes (escribir en ygg via API)
```
